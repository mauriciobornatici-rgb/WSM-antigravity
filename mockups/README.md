# Mockups POS y Alta de Producto

Archivos generados para analisis (sin tocar la app productiva):

- `mockups/pos-visual-a.html`
- `mockups/pos-compacto-b.html`
- `mockups/alta-producto-escaneo-c.html`

Assets:

- `mockups/assets/products/*.svg`

## Como abrir

En Windows, abrir directamente con doble click o desde terminal:

```powershell
start .\mockups\pos-visual-a.html
start .\mockups\pos-compacto-b.html
start .\mockups\alta-producto-escaneo-c.html
```

## Objetivo de cada variante

1. POS Visual A: validacion visual rapida por foto de producto.
2. POS Compacto B: mayor velocidad operativa con tabla densa.
3. Alta Producto C: escaneo de codigo, carga de imagen y escaneo de ubicacion.

## Criterios para elegir

- Tiempo promedio de carga de item a carrito.
- Tasa de error por seleccion incorrecta.
- Claridad para nuevos cajeros.
- Rendimiento en pantallas de baja resolucion.
- Flujo real con lector optico y telefono.
